# Система совместной разработки (хакатон)

## Описание проекта
Данный проект представляет собой систему управления предложениями для совместной разработки.  
Пользователи могут создавать, просматривать, обсуждать и голосовать за предложения, а администраторы управляют настройками системы.

## Основной функционал
- **Создание предложений** — добавление новых идей через веб-интерфейс.
- **Просмотр предложений** — список предложений, разбитый по статусам: `Новые`, `В работе`, `Принятые`, `Отклонённые`.
- **Обсуждение** — комментарии и ответы на предложения.
- **Голосование** — поддержка системы голосования с возможностью отмены голоса.
- **Администрирование** — управление настройками системы и распределением голосов.

## Используемые технологии
- **Backend**: Python (FastAPI)
- **Database**: PostgreSQL
- **Containerization**: Docker, Docker Compose
- **Version Control**: GitHub

## Развёртывание проекта (Docker)
Для быстрого запуска проекта используется **Docker Compose**.

1. **Установите зависимости (если необходимо)**
Убедитесь, что на вашей системе установлены:
- **Docker**
- **Docker Compose**
2. **Запуск проекта**
Запустить backend данных можно командой:
```
docker-compose up --build
```
После успешного запуска API будет доступен по адресу:
[http://localhost:8000]

## Остановка контейнеров
Чтобы остановить и удалить контейнеры, используйте:
```
docker-compose down
```
